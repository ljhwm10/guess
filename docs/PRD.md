# Draw & Guess(你画我猜)产品需求文档(PRD)

> 版本:1.0 · 日期:2026-07-13 · 状态:已定稿(自动 brainstorming 产出)
> 参考产品:玩吧「你画我猜」、Steam「Draw & Guess」、skribbl.io

## 1. 产品概述

一个免登录、多人实时在线的你画我猜游戏。玩家输入昵称即可建房或加入房间,轮流作画,其余玩家通过聊天框猜词得分,游戏结束展示排名。支持手机与电脑同房间游玩,支持房间内语音通话。

## 2. 目标用户与场景

- 朋友/同事聚会时的快速休闲游戏:一人建房,其他人手机扫码或输入房号加入。
- 无需注册登录,打开即玩,单局 5~15 分钟。

## 3. 功能需求

### 3.1 进入游戏(P0)

- 首页输入昵称(1~12 字符,去首尾空格,必填)即可进入大厅。
- 昵称与玩家身份(playerId,UUID)持久化在浏览器 localStorage,刷新/断线后可恢复身份。
- 无需任何账号体系。

### 3.2 房间系统(P0)

- **创建房间**:房主可配置
  - 人数上限:2~8 人(默认 8);
  - 轮数:1/2/3 轮(默认 2,每轮每位玩家各画一次);
  - 每回合作画/猜词时长:30~180 秒按 10 秒步进(默认 90);
  - 类型提示延迟:0~60 秒按 5 秒步进(默认 20;0 = 开局即显示);
  - 候选词数量:3/4/5 个(默认 3)。
- 房间生成 6 位数字房号;首页展示公开房间列表(房名=房主昵称、人数、状态),也可输入房号直接加入。
- 仅**等待中**(лobby)的房间可加入;游戏进行中的房间不可中途加入。
- 房主退出后,房主身份自动移交给最早进房的其他玩家;房间无人后销毁。

### 3.3 准备与开始(P0)

- 非房主玩家可点击「准备/取消准备」。
- 房主不需要准备;当**人数 ≥ 2 且其他玩家全部已准备**时,房主可点击「开始游戏」。
- 房间内实时展示每位玩家的准备状态。

### 3.4 游戏流程(P0)

1. 按进房顺序轮流作画,`总回合数 = 轮数 × 玩家数`。
2. 每回合:
   - **选词阶段(15 秒)**:画者从配置数量(3/4/5)的候选词(含分类提示)中选 1 个;可点击「换一批」刷新候选词列表(每回合限 1 次,不重置倒计时,新一批与已展示过的不重复);超时自动选当前列表第一个。
   - **作画阶段(30~180 秒,默认 90)**:画者作画,其他玩家在聊天框猜词。
   - 猜词者提示 = 字数 + 类型:开局仅显示字数(下划线);**类型提示在开局 categoryHintSeconds 秒后(默认 20 秒)才显示**;剩余时间 < 40% 时另揭示其中一个字。
   - 全部人猜中则回合提前结束;回合间展示答案与本回合得分(5 秒)。
3. 游戏结束:展示最终排名(前三名奖牌样式),支持「再来一局」——全员回到房间且重置准备状态。

### 3.5 计分规则(P0)

- 猜中者得分:`max(10, round(100 × 剩余时间/总时长))`;第一个猜中者额外 +20。
- 画者得分:每有一人猜中 +25。
- 无人猜中画者不得分。

### 3.6 画板(P0)

- **比例一致**:所有设备上画板逻辑坐标固定为 800×600(4:3),按屏幕等比缩放显示,笔画坐标归一化传输,保证手机和电脑看到的画面构图完全一致。
- 工具:
  - 画笔:8 种颜色(黑/红/橙/黄/绿/蓝/紫/棕),3 档粗细;
  - 橡皮擦:白色大笔刷,3 档粗细;
  - 清空画板(仅画者,需二次确认防误触)。
- 手机支持触摸作画,电脑支持鼠标作画;仅当前画者可操作画板。
- 观看端实时(增量)看到笔画,中途重连可恢复整幅画面。

### 3.7 实时聊天与猜词(P0)

- 聊天框输入即猜词(WebSocket 实时);猜中时:
  - 本人看到「猜对了!」并显示答案,之后的发言仅对画者与已猜中玩家可见(防剧透);
  - 其他人看到系统消息「××× 猜对了」;
  - 答案接近(包含答案或答案包含其输入,且未全中)时私下提示「很接近了!」。
- 画者不能发送包含答案的消息(直接拦截提示)。
- 等待/结算阶段聊天为普通公开聊天。

### 3.8 语音通话(P1)

- 房间内可选加入语音(WebRTC P2P,≤8 人 mesh);支持麦克风开/关(静音)。
- 显示房间内各成员语音状态(未加入/已加入/静音)。
- 约束:浏览器要求安全上下文(HTTPS 或 localhost)才能取麦克风;不满足时界面降级提示,不影响游戏其他功能。
- 作画者语音不禁用(与玩吧一致,靠自觉;聊天猜词才计分)。

### 3.9 断线与容错(P0)

- 玩家断线后保留 60 秒,期间头像置灰;同一浏览器重连(playerId 匹配)自动恢复到原座位、原分数,并同步当前画面与游戏进度。
- 画者掉线:回合立即结束(公布答案),游戏继续。
- 游戏中人数(含离线)< 2:游戏终止,回到房间等待界面。

## 4. 非功能需求

- 手机(375px 宽)与桌面(≥1024px)均可完整操作;画板区域始终保持 4:3。
- 笔画同步延迟 < 200ms(局域网);单房间消息广播 O(人数)。
- 服务重启后房间数据丢失可接受(内存态,无持久化)。
- 中文界面。

## 5. 页面清单

| 页面 | 内容 |
|---|---|
| 首页 | 昵称输入、创建房间(配置弹层)、房间列表、房号加入 |
| 房间(等待) | 玩家列表+准备状态、房间配置展示、准备/开始按钮、聊天、语音控制 |
| 游戏 | 画板+工具栏(画者)、词提示条、倒计时、玩家侧栏(分数/状态)、聊天猜词、语音控制 |
| 结算 | 名次列表(前三奖牌)、再来一局/离开 |

## 6. 验收标准

1. 两个浏览器(其一为手机模拟 viewport)可完成:输入昵称 → 建房 → 加入 → 准备 → 开始 → 选词 → 作画/猜词 → 计分 → 多回合 → 结算排名 → 再来一局 的完整闭环。
2. 画板在 375px 手机与桌面上显示比例一致,笔画位置一一对应。
3. 断开其中一个客户端网络后 60 秒内刷新可恢复进行中的对局。
4. 服务端游戏逻辑单元测试与 socket 集成测试全部通过。
